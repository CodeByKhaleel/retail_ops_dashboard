# Security Policy

## Supported Versions

The latest version of this project is actively maintained and receives security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please avoid creating public issues for sensitive vulnerabilities.

Instead, contact:

* GitHub Issues (for non-sensitive concerns)
* Direct maintainer communication for critical vulnerabilities

---

## Security Best Practices

This project follows security best practices including:

* Environment variable protection
* Input validation
* Dependency monitoring
* Secure authentication practices
* Rate limiting where applicable

---

## Dependency Security

Dependencies should be kept updated regularly to reduce security risks.

Please ensure:

* `npm audit` is reviewed periodically
* vulnerable packages are updated promptly

---

Thank you for helping keep the project secure.
